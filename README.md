# AIカードバトラー

PCを相手にした、シンプルで見た目も楽しい1対1のカードバトルゲームです。プレイヤーはユニークな画像と能力を持つカードデッキを使い、相手の体力を0にすることを目指します。

## GitHub Pagesへのデプロイ方法

このプロジェクトをWebサイトとして公開（デプロイ）するには、あなたのコンピュータ上で「ビルド」という作業を行う必要があります。これにより、ブラウザが直接実行できる形式のファイル群が生成されます。

### 準備：フォルダ構成の変更（一度だけ）

ビルドプロセスが画像ファイルを正しく認識できるように、一度だけフォルダの構成を変更します。

1.  プロジェクトのルートディレクトリ（`index.html`がある場所）に、`public`という名前の新しいフォルダを作成します。
2.  既存の`Image2`フォルダを、作成した`public`フォルダの中に移動させます。

最終的なフォルダ構成は以下のようになります。

```
/
├── public/
│   └── Image2/
│       ├── 1.jpeg
│       └── ...
├── components/
├── App.tsx
├── index.html
└── ... (その他のファイル)
```

### ビルドとデプロイの手順

1.  **ターミナル（コマンドプロンプトやPowerShellなど）を開き、このプロジェクトのフォルダに移動します。**

2.  **依存関係のインストール:**
    プロジェクトに必要なライブラリをインストールします。このコマンドは、最初に一度だけ実行すればOKです。
    ```bash
    npm install
    ```

3.  **プロジェクトのビルド:**
    以下のコマンドを実行して、デプロイ用のファイルを生成します。
    ```bash
    npm run build
    ```
    成功すると、プロジェクト内に`dist`という名前のフォルダが新しく作成されます。このフォルダには、Webサイトとして公開するために最適化されたHTML, CSS, JavaScript, 画像ファイルなどがすべて含まれています。

4.  **GitHubリポジトリへのアップロード:**
    **`dist`フォルダの中に生成された全てのファイルとフォルダ**を、あなたのGitHubリポジトリにアップロード（プッシュ）してください。
    **注意:** `dist`フォルダ自体をアップロードするのではなく、**その中身**をリポジトリのルートに配置します。

5.  **デプロイの確認:**
    ファイルをアップロードした後、数分待ってからあなたのGitHub PagesのURLにアクセスしてください。ゲームが正しく表示されるはずです。

    （例: `https://<あなたのユーザー名>.github.io/<リポジトリ名>/`）

---

### よくある質問

**Q: なぜビルドが必要なのですか？**
A: GitHub Pagesのサーバーは、開発用のファイル形式（`.tsx`）を直接理解できません。ビルドプロセスによって、これらのファイルをブラウザが直接実行できる標準的なJavaScript（`.js`）形式に変換する必要があるためです。
